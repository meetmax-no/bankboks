## 4. Reserverte subdomener

```typescript
const RESERVED_SUBDOMAINS = [
  "admin", "api", "app", "www", "mail", "ftp",
  "smtp", "imap", "pop", "start", "test", "sub",
  "register", "platform", "billing", "support",
  "help", "status", "health", "demo", "dev",
  "staging", "prod", "beta", "alpha", "sandbox",
  "kodo", "kodovault", "vault", "michael", "mike"
]

function isReservedSubdomain(subdomain: string): boolean {
  if (RESERVED_SUBDOMAINS.includes(subdomain)) return true
  if (subdomain.endsWith("-admin")) return true
  return false
}
```

## 5. Registreringsflyt

### FLYT 1 — Gratis trial

```
Bruker → /register?plan=trial
    ↓
Registreringsskjema:
    • Fornavn (valgfritt)
    • Etternavn (valgfritt)
    • E-post (påkrevd)
    • Subdomene (sanntidssjekk)
    • ☑ Send meg påminnelser om prøveperioden og betalingsstatus (anbefalt)
    • Cloudflare Turnstile (usynlig)
    ↓
POST /api/register
    1. Verifiser Turnstile token
    2. Rate-limit (maks 2 per IP per 24t)
    3. Valider subdomene (format + reservert + ledig)
    4. Valider e-post (format)
    5. Skriv TenantRecord { status: "trial" }
    6. Generer clients/<subdomain>.json
    7. Kall Vercel API → opprett prosjekt
    8. Kall Vercel API → koble subdomene
    9. Kall Upstash API → provisjoner instans
    10. Send velkomstmail via Resend
    ↓
Skjerm 6: Ferdig → terje.kodovault.no
```

### FLYT 2 — Betalt plan

```
Bruker → /register?plan=monthly (eller yearly)
    ↓
Registreringsskjema (identisk med trial)
    ↓
POST /api/register/paid
    1. Verifiser Turnstile + rate-limit + validering
    2. Reserver subdomene { status: "pending", expiresAt: now + 30min }
    3. Opprett Stripe customer
    4. Opprett Stripe Checkout session med metadata
    ↓
Stripe Checkout
    ↓
Stripe webhook → /api/webhook
    1. Verifiser Stripe signatur
    2. Les metadata
    3. Skriv TenantRecord { status: "active" }
    4. Generer clients/<subdomain>.json
    5. Kall Vercel API → opprett prosjekt
    6. Kall Vercel API → koble subdomene
    7. Kall Upstash API → provisjoner instans
    8. Send velkomstmail + betalingsbekreftelse
    ↓
Skjerm 5: Provisjonering (polling /api/status)
    ↓
Skjerm 6: Ferdig → terje.kodovault.no
```

### FLYT 3 — B2B Enterprise (Mike-styrt)

```
Mike → admin.kodovault.no
    ↓
POST /api/admin/create-enterprise
    1. Autentiser Mike (admin.kodovault.no + gyldig session)
    2. Valider alle felter
    3. Opprett Stripe customer + send faktura
    4. Skriv TenantRecord { customerType: "b2b" }
    5. Generer clients/<adminSubdomain>.json
    6. Kall Vercel API + Upstash API
    ↓
am-admin.kodovault.no er live
```

## 7. Livssyklus — Cron

```
Vercel Cron: daglig kl. 02:00 UTC → /api/cron/lifecycle

Dag 25 (lifecycle: true):
    → E-post: "5 dager igjen" + betalingslink
    → Telegram til Mike

Dag 30 (ikke betalt):
    → status: "locked"
    → E-post: "Vault låst" (transaksjonell)
    → Telegram til Mike

Dag 37 (lifecycle: true):
    → E-post: purring + kortbetalingsalternativ

Dag 44:
    → status: "locked" (hard lukking)
    → E-post: "Vault lukket" (transaksjonell)

Dag 51 (lifecycle: true):
    → E-post: "Sletter om 7 dager"

Dag 58:
    → Slett Vercel-prosjekt
    → Slett Upstash-instans
    → status: "deleted"
    → E-post: "Din vault er slettet" (transaksjonell)
    → TenantRecord beholdes i 2 år
    → Telegram til Mike

Vercel Cron: hvert 30. min → /api/cron/cleanup-pending
    → Finn alle { status: "pending", expiresAt < now }
    → Slett reservasjon
```

## 8. Feilscenarier

```
Turnstile feiler:
    → Blokker, vis feilmelding, logg

Sentral Upstash nede:
    → Blokker registrering, vis feilmelding

Stripe session feiler:
    → Slett pending-reservasjon, vis feilmelding

Stripe webhook aldri mottatt:
    → Cron rydder pending > 30 min automatisk

Stripe webhook ugyldig signatur:
    → Avvis (HTTP 400), logg

Vercel API feiler:
    → status: "provisioning_failed"
    → Telegram + e-post til Mike
    → Bruker ser: "Vi setter opp vaulten din..."
    → Retry 3 ganger med 60 sek mellomrom

Upstash provisjonering feiler:
    → Samme som Vercel API feil
    → Rull tilbake Vercel-prosjekt

Cron feiler:
    → Vercel logger automatisk
    → Datoberegning basert på createdAt — ikke antall kjøringer
    → Neste kjøring tar opp der det slapp

B2B faktura feiler:
    → status: "invoice_failed"
    → Telegram + e-post til Mike
    → Enterprise-vault forblir aktiv til Mike låser manuelt
```

## 11. Sikkerhet

```
Cloudflare Turnstile:     usynlig bot-filter på /register
Rate-limiting:            maks 2 registreringer per IP per 24t
Reserverte subdomener:    liste + endsWith("-admin") blokkert
Stripe webhook-signatur:  verifiseres før alle handlinger
Admin-autentisering:      kun admin.kodovault.no + gyldig session
Sentral Upstash:          kryptert AES-256-GCM

Vercel env-vars:
    STRIPE_SECRET_KEY
    STRIPE_WEBHOOK_SECRET
    TELEGRAM_BOT_TOKEN
    TELEGRAM_CHAT_ID
    RESEND_API_KEY
    TURNSTILE_SECRET_KEY
    CENTRAL_UPSTASH_URL
    CENTRAL_UPSTASH_TOKEN
```
